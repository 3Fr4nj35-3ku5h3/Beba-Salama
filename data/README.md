# Data Dictionary

## Layer 1 — World Bank / Ma3Route Kenya Road Traffic Crashes (2012–2023)

**Files:** `raw/ma3route_crashes_algorithmcode.csv` (31,064 rows), `raw/ma3route_crashes_manualcode.csv` (2,595 rows, manually verified subset, Jul 2017–Jul 2018)

Geolocated from crowdsourced reports to `@Ma3Route`, a platform that crowdsources Kenyan transport data via Twitter.

| Column | Type | Description |
|---|---|---|
| `crash_id` | int | Unique crash identifier |
| `crash_datetime` | datetime | Date/time of first tweet reporting the crash |
| `crash_date` | date | Date only |
| `latitude` / `longitude` | float | GPS coordinates (primarily Nairobi) |
| `n_crash_reports` | int | Number of tweets reporting this crash — proxy for severity/visibility |
| `contains_fatality_words` | binary | Tweet contains "dead", "died", "body", "killed", or "fatal" |
| `contains_pedestrian_words` | binary | Tweet contains "pedestrian" |
| `contains_matatu_words` | binary | Tweet contains "matatu" |
| `contains_motorcycle_words` | binary | Tweet contains "boda", "motorcycle", or "motor cycle" |

**Full citation requirement:** Milusheva S, Marty R, Bedoya G, Williams S, Resor E, et al. (2021) "Applying machine learning and geolocation techniques to social media data (Twitter) to develop a resource for urban planning." *PLOS ONE* 16(2): e0244317. https://doi.org/10.1371/journal.pone.0244317

Source: [microdata.worldbank.org/index.php/catalog/6249](https://microdata.worldbank.org/index.php/catalog/6249) — see `raw/ddi-documentation-english_microdata-6249.pdf` for the full DDI documentation.

## Layer 2 — Kenya Police / HDX Road Accidents Database (2016–2017)

**File:** `raw/kenya-accidents-database.xlsx` — two sheets, `2016` (378 rows, Apr–Jun) and `2017` (740 rows, Feb–Nov). 1,118 records total, ~1,316 victims.

| Column | Description |
|---|---|
| `Date DD/MM/YYYY` | Date of incident |
| `TIME 24 HOURS` | Hour of incident |
| `COUNTY` | County where incident occurred |
| `ROAD` | Road name (⚠️ inconsistent — see Known Issues) |
| `PLACE` | Specific location on that road |
| `CAUSE CODE` | Coded cause (legend not included in source file — cross-reference with NTSA classification if available) |
| `VICTIM` | Victim type (pedestrian, motorcyclist, passenger, etc.) |
| `NO.` | Number of victims in that incident |
| `GENDER` / `AGE` | Victim demographics |
| `BRIEF ACCIDENT DETAILS` | Free-text description |

⚠️ **License terms for this specific HDX resource were not located at time of writing.** Confirm on [data.humdata.org/dataset/kenya-road-accidents-database](https://data.humdata.org/dataset/kenya-road-accidents-database) before any redistribution beyond this project's academic use.

### Known issues
- **Road name inconsistency:** the same road appears under multiple spellings (e.g. `NAIROBI MOMBASA`, `MOMBASA-NAIROBI`, `MOMBASA ROAD` are all one road). Standardisation happens in `src/data_prep.py::standardize_road_names()`.
- **No severity label:** there is no clean fatal/injury/minor column — a severity proxy must be engineered (see `src/features.py`).
- **Partial calendar coverage:** 2016 covers 3 months only; 2017 covers 10. Seasonal comparisons across this layer alone are not reliable.
